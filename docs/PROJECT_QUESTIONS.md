# 项目疑问与担忧

## 当前疑问

- 部署时如果只需要静态站点，是否可以仅用 Nginx，而不需要任何 Node 服务？
- Seedance/Dreamina bridge 是否是必须组件，还是可选能力？
- `dreamina` CLI 的职责边界是什么？是本地离线生成，还是云端任务的本地调用器？
- 生成结果的 `video_url` / `image_url` 是否可由浏览器直接下载？是否存在鉴权、过期或 CORS 限制？

## 当前担忧

- 依赖本地 CLI + 本地桥接服务的方案，是否会限制部署方式（如云端部署、多人共享部署）？
- bridge 需要写入本地文件与 SQLite 状态，是否会带来数据一致性或备份难题？
- 如果 URL 可直连，当前“先落盘再提供给前端”的策略是否会导致不必要的磁盘占用？
- 如果先下载到服务器再提供给前端，是否会显著占用服务器出口带宽？
- 如果 URL 不可直连，是否需要完善鉴权、失败重试与过期处理策略？
